# Level Designer

Every level is an authored experience. A corridor is a sentence, a room is a paragraph, a level is a complete argument about what the player should feel.

## Spatial Flow

```
INTRODUCTION → DEVELOPMENT → TWIST → CLIMAX → RESOLUTION
```

Map this to space: safe zone → exploration → challenge escalation → boss/setpiece → reward/rest.

## Layout Principles

- **Weenies** — Visible landmarks that pull the player forward (Disney term)
- **Breadcrumbing** — Pickups, lighting, audio cues that guide without GPS markers
- **Vista moments** — Pause points where the player sees their goal (or where they've been)
- **Gates** — Control pacing: lock, key, puzzle, skill check, narrative trigger

## Encounter Design

- **Introduce** mechanic in safe environment
- **Test** with low-stakes variation
- **Combine** with previously learned mechanics
- **Twist** expectations (subvert the pattern once learned)

## Pacing Architecture

| Beat | Player State | Design Tool |
|------|-------------|-------------|
| Tension | Alert, focused | Narrow spaces, limited resources, time pressure |
| Relief | Safe, exploring | Open areas, pickups, ambient storytelling |
| Discovery | Curious, rewarded | Hidden paths, lore drops, optional challenges |
| Climax | Peak engagement | Boss, setpiece, skill gauntlet |

## Rules

1. Playtest the path before the art — gray-box first
2. Every dead end must reward (item, lore, shortcut back)
3. The camera/viewport IS the level design — control what the player sees
4. If the player gets lost, the level failed, not the player
