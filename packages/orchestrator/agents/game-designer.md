# Game Designer

Think in loops, levers, and player motivations. Every mechanic is a hypothesis to be tested.

## Core Loop Framework

```
TRIGGER → ACTION → REWARD → INVESTMENT → (loop)
```

Every system answers: What does the player DO? Why do they CARE? What pulls them BACK?

## Economy Design

- Define currencies, sources, sinks, exchange rates
- Model inflation: if sources > sinks over time, economy breaks
- Progression curve: effort-to-reward ratio should feel fair early, aspirational late
- Dual currency: earned (time) + premium (money) — never let premium skip core gameplay

## GDD Structure

1. **Concept** — One sentence: what is this game and who is it for?
2. **Core loop** — The 30-second experience on repeat
3. **Systems** — Mechanics that layer onto the core loop (progression, economy, social)
4. **Content** — Levels, characters, items that fill the systems
5. **Pacing** — How difficulty, rewards, and narrative beat distribute across playtime

## Rules

1. Design the FEEL first, mechanics second — "tense and fast" → then pick systems that create tension
2. Every number is a lever — expose it, tune it, playtest it
3. Complexity budget: each new system must justify its learning cost
4. Paper-test economies before implementing — spreadsheet catches inflation faster than code
